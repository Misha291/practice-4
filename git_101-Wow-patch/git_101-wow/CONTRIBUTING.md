# Пошаговая инструкция для выполнения задания в GitVerse SmartClass

## Вариант 1. Через веб-интерфейс GitVerse

### 1. Создание новой ветки  
1. На странице своего учебного репозитория нажмите кнопку **Ветки**.  
2. В поле «Создать новую ветку» введите имя ветки, например `feature1`.  
3. Нажмите **Создать ветку**.  

### 2. Редактирование файла README.md  
1. В списке файлов кликните на `README.md`.  
2. Нажмите кнопку **Редактировать** (иконка карандаша).  
3. Добавьте любой текст, например «Первый коммит».  
4. Внизу страницы заполните поле «Комментарий» — кратко опишите изменения (например «Добавил приветствие»).  
5. Нажмите **Сохранить изменения**.
- *Примечание*: для прохождения автоматизированной проверки необходимо изменить не менее 10 строк в файле `README.md`. 
- *Примечание*: помимо отслеживания изменений в файле `README.md` будет также оцениваться длина комментариев к коммиту (не менее 10 символов и не более 100)

### 3. Создание новых коммитов  
1. Повторите редактирование `README.md` или любого другого файла: нажмите **Редактировать**, сделайте правку.  
2. Придумайте осмысленное сообщение, например «Обновил описание проекта», и нажмите **Сохранить изменения**.
- **Примечание**: для прохождения автоматизированной проверки необходимо чтобы в новой ветке было не менее 5 коммитов

### 4. Создание Запроса на слияние  
1. После того как нужное количество коммитов будет создано необходимо сделать Запрос на слияние.  
2. В панели gitverse выбираем «Запросы», затем нажимаем на **Создать новый запрос**.
3. В новом окне выбираем ветку источник (например `feature1`) и нажмите **Создать запрос**
- *Примечание*: рамках текущего задания нет ограничений на содержание комментария к запросу на слияние, однако на практике необходимо их тоже делать подробными и понятными

### 5. Автоматическая проверка задания
1. После создания запроса на слияния запустится механизм автопроверки, за которым можно проследить выбрав на панели элемент «Запросы».
Cистема автоматически проверит:
- Есть ли новая ветка (1 балл)  
- Редактирован ли файл README.md (1 балл)  
- Выполнено ли минимум два коммита (1 балл)  
- Используются ли качественные сообщения коммитов (1 балл)  
- Создан ли Pull Request (1 балл) 
2. В случае если какое-то из заданий оказалось не сделанным то на шаге **Autograding Reporter** всплывет красный знак с восклицательным знаком.
  1. Нажмите на **Autograding Reporter** можно будет увидеть какие из элементов задания оказались невыполнеными. 
  2. Устраните возникшие замечания.
3. В случае если все задания выполнены корректно то во всех пунктах автопроверки должны загореться зеленые галочки и задание будет считаться выполненым.
4. После успешного выполнение задания введите свой ник в качетсве ответа на задание **Основы git** на платформе elearn.urfu.ru
